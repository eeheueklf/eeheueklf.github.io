# Noiton

 
인원 | 개인 프로젝트  
기간 | 2026.01 - 2026.03  
기술 | Next.js, TypeScript, TailwindCSS, Supabase

> **복잡한 노션 템플릿을 자유롭게 공유하고 체계적으로 탐색하는 플랫폼**  
> 단순한 파일 공유를 넘어, 생산성 노하우를 영감으로 주고받는 커뮤니티 공간을 지향합니다.

---

## 주요 기능

* 인기 템플릿 조회 및 카테고리별 탐색
* 최신/인기순 필터링 및 템플릿 검색
* 사용자 맞춤 템플릿 업로드 및 공유
* 템플릿 관리 *(찜한/업로드한 템플릿 확인)*
* 구글 OAuth 간편 로그인 및 프로필 설정

## 개발 이슈 및 해결 과정

1. **초기 렌더링 성능 최적화 (LCP 75% 개선) 🚨**
    * `issue`: 메인 페이지 진입 시 대량의 템플릿 썸네일 이미지와 검색 바 리소스가 한꺼번에 로드되면서 지연이 발생했습니다. 이로 인해 최대 콘텐츠 렌더링 시간 [LCP](https://web.dev/articles/lcp?hl=ko)가 9.8초까지 지연되었습니다
    * `solve`: 리소스 로딩 우선순위를 전략적으로 설계하여 [중요 렌더링 경로](https://developer.mozilla.org/ko/docs/Web/Performance/Guides/Critical_rendering_path)를 최적화했습니다.
       * 뷰포트 상단 핵심 이미지는 `priority`로 프리로드하고, `sizes`와 `quality`를 조절해 페이로드 크기를 최적화했습니다.
       * 초기 렌더링에 필수적이지 않은 검색바는 `next/dynamic`로 분리하여 순서를 조정하였습니다.
    * `result`: `LCP` 지표를 9.8초에서 2.4초로 약 **75% 단축**하여 쾌적한 첫 화면 진입 속도를 확보했습니다.
        ```tsx 
        // 이미지 최적화
        <Image 
            ...
            quality={50} // 이미지 페이로드 최적화
            sizes="(max-width: 768px) 100vw, 300px" // 뷰포트 맞춤
            priority={index < 3} // 핵심 이미지 우선 로드
        />
        // JS 최적화 : 검색바 지연 로딩
        const HomeSearchBar = dynamic(() => 
            import("@/components/search/SearchBar").then(mod => mod.HomeSearchBar), 
            { 
                ssr: true, 
                loading: () => <div className="h-[46px] w-full max-w-[320px] bg-gray-100 animate-pulse rounded-full" /> 
            }
        )
        ```

2. **Next.js & Supabase 기반의 효율적인 데이터 아키텍처**
    * `issue`: 클라이언트 중심의 인증 처리와 데이터 가공은 보안 취약성을 높이고, JS 번들 용량 증가로 브라우저의 연산 부담을 가중시켰습니다.
    * `solve`:  `Supabase`와 서버 컴포넌트를 결합하여 보안과 성능을 동시에 확보했습니다.
        * 미들웨어에서 세션을 관리하여 클라이언트 노출 없이 보안이 강화된 인증 흐름을 구현했습니다.
        * API 시크릿 키와 무거운 데이터 파싱 로직을 서버에서만 실행하여, 클라이언트로 전송되는 JS 페이로드를 최소화했습니다.
    * `result`: 비즈니스 로직의 복잡도와 무관하게 일관된 로딩 성능을 유지할 수 있는 구조를 확립하고, 보안 안정성을 강화했습니다.

3. **URL 기반 상태 관리와 Breadcrumb** [관련 링크](https://noitoner.vercel.app/templates/personal/lifestyle/productivity)
    * `issue`: 카테고리가 복잡해질수록 사용자가 "내가 지금 어디쯤 있는지" 길을 잃기 쉬웠습니다. 또한, 새로고침을 하거나 링크를 복사해서 공유하면 선택했던 카테고리 필터가 유지되지 않아 처음부터 다시 찾아야 하는 불편함이 있었습니다.
    * `solve`: 데이터를 브라우저 메모리가 아닌 **주소창(URL)**에 직접 기록하는 방식을 선택했습니다.
        * 주소창이 데이터 저장소: "현재 위치와 필터 정보"를 주소(URL)에 담아, 별도의 복잡한 도구 없이도 주소 자체가 하나의 데이터가 되도록 설계했습니다.
        * 직관적인 경로 표시: 사용자가 거쳐온 단계를 '브레드크럼(Breadcrumb)' 형태로 보여주어, 클릭 한 번으로 상위 카테고리에 쉽게 돌아갈 수 있게 했습니다.
    * `result`: 별도의 전역 상태 관리 도구 없이도 공유 가능한 URL 구조를 완성했으며, 사용자는 어떤 링크를 통해서도 의도한 화면을 즉시 마주할 수 있게 되었습니다.
        ```tsx
            // URL 쿼리 스트링 활용
            export function CategoryNav({ items, basePath = "/templates" }) {
            return (
                <nav className="flex items-center gap-2 text-sm text-gray-400">
                {items.map((bc, index) => (
                    <React.Fragment key={bc.path}>
                    {index > 0 && <ChevronRight size={12} />}
                    <Link 
                        href={`${basePath}?category=${bc.path}`}
                        className={index === items.length - 1 ? "text-black font-medium" : ""}
                    >
                        {bc.name}
                    </Link>
                    </React.Fragment>
                ))}
                </nav>
            );
            }
            ```
