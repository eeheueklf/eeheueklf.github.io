
# TYLE

 
인원 | 2인 프로젝트  
기간 | 2025.08 - 2025.10  
기술 | Next.js, TypeScript, TailwindCSS, Hangul.js, Web Audio API
 
> **"오직 타이핑에만 몰입할 수 있는 환경"**   
> 사용자의 타수와 정확도를 실시간으로 측정하여 시각화하는 한국어 타자 연습 어플리케이션입니다. 잦은 상태 변화와 오디오 재생 등 브라우저 부하가 높은 환경에서도 끊김 없는 사용자 경험을 제공하는 데 집중했습니다.

---


## 주요 기능

* 한글 최적화 타이핑 엔진 및 실시간 타수 연산
* 타이핑 입력과 실시간으로 동기화되는 타자기 효과음
* '오늘의 글' 및 커스텀 연습 시스템
* 데이터 시각화 기반의 타자 기록 통계 및 공유

## 개발 이슈 및 해결 과정

1. **실시간 타수(CPM) 연산 최적화🚨** [관련 포스팅](docs/dev-log/tyle-performance)
    * `issue`: 한글 특성상 정확한 타수 계산을 위해 매 입력마다 자소 분리 로직이 실행됩니다. 특히 실시간 `CPM` 표시를 위해 `100ms` 단위의 타이머가 동작하면서 입력이 없는 순간에도 무의미한 연산이 반복되어 텍스트가 길어질수록 입력 지연이 발생했습니다.
    * `solve`: 연산 결과의 재사용성을 높이고 불필요한 리렌더링 부하를 줄였습니다.
        * 타이머 생명주기 관리: setInterval 사용 시 발생할 수 있는 메모리 누수를 방지하기 위해 useEffect의 Cleanup 함수를 활용, 컴포넌트 언마운트 및 연습 종료 시점에 프로세스를 확실히 종료하도록 설계했습니다.
        * 연산 메모이제이션: useMemo를 도입하여 타이머 상태와 상관없이 실제 입력값(inputValue)이 변할 때만 타수 계산 로직이 실행되도록 캐싱했습니다.
        ```tsx
        // 입력 데이터의 변화가 있을 때만 한글 자소 분리 및 타수 계산 수행
        const memoizedCPM = useMemo(() => {
          if (!currentInput) return 0;
          const jasoCount = Hangul.disassemble(currentInput).length;
          return calculateCPM(jasoCount, startTime);
        }, [currentInput, startTime]); // 입력값 변경 시에만 트리거
        ```
    * `result`: 리렌더링 시 발생하는 불필요한 연산 횟수를 획기적으로 줄여, 긴 문장 입력 시에도 CPU 점유율을 안정적으로 유지하고 끊김 없는 타이핑 환경을 구축했습니다.
    
2. **Web Audio API를 활용한 저지연(Low-Latency) 사운드 구현**
    * `issue`: 기존 `audio` 태그 방식은 브라우저의 오디오 스트리밍 처리 한계로 인해 빠른 타이핑 속도를 따라오지 못하는 사운드 밀림 현상이 발생했습니다. 이는 시각적 입력과 청각적 피드백의 불일치를 유발하여 사용자 경험을 저해했습니다.
    * `solve`: 비동기 오디오 처리에 특화된 Web Audio API로 전환했습니다.
        * 오디오 버퍼링: 효과음 데이터를 메모리에 미리 로드(Pre-load)하여 재생 시점의 디스크 I/O를 제거했습니다.
        * 비동기 재생: 메인 렌더링 스레드와 분리된 비동기 방식으로 사운드를 재생하여, 고속 타이핑 시에도 연산 성능에 영향을 주지 않고 즉각적인 피드백을 구현했습니다.
    * `result`: 오디오 지연 시간을 0에 가깝게 구현하여, 초당 10타 이상의 빠른 입력에서도 실제 타자기를 치는 듯한 리드미컬하고 몰입감 있는 사용자 경험을 완성했습니다.

3. **웹 성능 지표 CLS 개선 (0.137 → 0)** [관련 포스팅](docs/dev-log/tyle-cls)
    * `issue`: 페이지 로드 초기, 커스텀 키보드 컴포넌트의 복잡한 DOM 구조와 스타일 연산으로 인해 하이드레이션이 지연되었습니다. 이 과정에서 브라우저가 하단 영역의 높이를 인지하지 못해 상단 콘텐츠가 아래로 갑자기 밀리는 Layout Shift(레이아웃 이동) 현상이 발생했습니다.
    * `solve`: 콘텐츠가 로드된 후 레이아웃을 침범하는 것이 아니라, 이미 확보된 자리에 스며드는 "선 점유, 후 렌더링" 전략을 채택했습니다.
        * 고정 높이 스켈레톤 설계: 실제 키보드 컴포넌트가 렌더링되었을 때의 높이(236px)를 분석하여, 이와 동일한 높이 값을 가진 Skeleton UI를 제작했습니다.
        * 선언적 로딩 처리(Suspense): Next.js Suspense를 도입하여, 키보드 리소스(자모 분리 로직, 사운드 등)가 준비되는 동안 스켈레톤이 영역을 견고하게 방어하도록 설계했습니다
    * `result`: 하이드레이션 전후의 레이아웃 변화를 원천 차단하여 Lighthouse 측정 기준 CLS 점수 0을 달성했습니다. 페이지 새로고침 시에도 UI 흔들림이 없는 안정적인 렌더링 환경을 구축했습니다.
